hexo deploy 时失败:Host key verification failed.

原因：
之前把dsa密钥换成rsa,github和gitee没换，上两个仓库添加之
还是不行，于是查看代码，看得头昏脑涨也没找到原因
后来灵光一现，用命令行clone一下，发现要手动确认
Are you sure you want to continue connecting (yes/no)? yes
hexo的错误提示没半点作用
又搜了下
ssh 登录出现Are you sure you want to continue connecting (yes/no)?解决方法
1,可以使用ssh -o 的参数进行设置
例如: ssh -o StrictHostKeyChecking=no root@192.168.111.22

如果在hexo提交的位置加上-o选项应该能解决这个问题，不过目前还没找到hexo提交的源码位置
